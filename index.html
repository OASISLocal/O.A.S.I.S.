Privacy first

Your prompts never
leave your machine.

No telemetry. No training on your data. No account required. OASIS is software you own, not a service you rent.

Fully offline inference
Run any supported model without an internet connection. Your prompts are processed entirely on your CPU or GPU — nothing is transmitted.
Local-only storage
Conversations, files, and settings live in a local database you control. Export, delete, or move your data whenever you want.
No telemetry, ever
OASIS collects zero usage data. There are no analytics, no crash reports sent without consent, and no subscription server pinging home.
You pick the connections
Each skill that needs network access — web search, remote APIs — requires an explicit toggle. Everything is off by default.

Common questions

Before you download.

Hardware requirements, model compatibility, and privacy — answered.

A GPU speeds things up significantly but is not required. On a modern CPU, a 7–8B model typically generates 5–15 tokens/second — fast enough for most conversational use. If you have a CUDA-capable NVIDIA GPU or Apple Silicon (M1/M2/M3), OASIS will automatically offload layers to it for much better throughput.

The recommended starting point is an 8 GB RAM machine running llama3.2:8b in CPU-only mode. It works. It's just not instant.

OASIS natively supports .gguf files via a bundled llama.cpp backend. If you have Ollama installed, OASIS can connect to it directly and use any model you've already pulled — no re-download needed.

Safetensors, GGML (legacy), and GPTQ are not currently supported as local backends, though remote API endpoints accept any format the provider serves.

A rough rule: the model file size in GB ≈ the RAM required, plus about 1–2 GB overhead for OASIS itself. An 8B model in Q4 quantization typically runs in 5–6 GB of RAM. With 8 GB total you're fine. With 16 GB you can comfortably run 13B models.

If you're tight on RAM, try phi-3-mini — it fits comfortably in 4 GB and still handles code, writing, and Q&A well.

No. When running a local model, inference happens entirely on your machine. OASIS has no telemetry, no analytics, and no update server that phones home. You can run it permanently in airplane mode.

The only exception is if you explicitly enable a remote endpoint (like the OpenAI API) under Settings → Models. In that case, your prompts go to that provider — which you've opted into. The toggle is clearly labeled and off by default.
